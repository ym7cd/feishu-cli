# 布局系统

feishu-cli 画板使用绝对坐标系统，没有自动布局引擎。所有节点的位置和大小必须手动计算。

## 布局决策

> 不要靠关键词猜布局。先分析信息结构，再决定布局策略。

| 判断条件 | 布局策略 | 方法 |
|----------|----------|------|
| 有明确上下层级（用户层 → 服务层 → 数据层） | **分层条带** | 每层 y 递增，层内 x 均分 |
| 网格矩阵（对比表、功能矩阵） | **行列对齐** | row * col 网格，固定间距 |
| 多个独立模块平级互联 | **岛屿式** | 手算每个模块位置，模块间用连线 |
| 树形层级（组织架构、模块依赖） | **树状展开** | 根居中，子节点向下展开 |
| 自由拓扑（鱼骨图、地图） | **自由定位** | 根据语义手算每个节点坐标 |
| 不确定 | **分层条带**（最安全） | 默认按行排列 |

| 布局策略 | 适用图表 |
|----------|----------|
| 分层条带 | 架构层级图、技术栈、流程图 |
| 行列对齐 | 对比表、功能矩阵、卡片墙、看板 |
| 岛屿式 | 系统集成图、微服务拓扑 |
| 树状展开 | 组织架构、模块依赖树 |
| 自由定位 | 鱼骨图、柱状图、折线图、飞轮图 |

---

## 标准间距参考

| 参数 | 推荐范围 | 说明 |
|------|---------|------|
| 整图宽度 | 800-1400px | 视内容量调整 |
| 层间距（行间距） | 80-120px | 不同层/行之间的 y 间距 |
| 同层节点间距 | 30-60px | 同一行内的 x 间距 |
| 有连线时最小间距 | >= 60px | 给箭头留空间 |
| 分区内边距 | 20-30px | 背景分区包裹子节点的 padding |
| 节点最小宽度 | 120px | 容纳基本文字 |
| 节点最小高度 | 40px | 单行文字 |

## 布局模式详解

### 分层条带（grid 模式）

最常用的布局。按行排列，每行 y 相同，适合架构图。

**计算方法**：

```
标题行: y = 20
第 1 层: y = 80
第 2 层: y = 80 + layer_height + 100 = 220
第 3 层: y = 220 + layer_height + 100 = 360

层内节点等间距排列:
  总宽度 = n * node_width + (n-1) * gap
  起始 x = (canvas_width - 总宽度) / 2
  第 i 个节点 x = 起始 x + i * (node_width + gap)
```

**示例：3 层架构图**（画布宽 800px）：

```
标题:        x=250, y=20,  w=300, h=40
背景-接入层: x=50,  y=70,  w=700, h=80   (z_index=0, opacity=25)
  节点1:     x=100, y=90,  w=140, h=40   (z_index=10)
  节点2:     x=280, y=90,  w=140, h=40
  节点3:     x=460, y=90,  w=140, h=40
背景-服务层: x=50,  y=170, w=700, h=80   (z_index=0, opacity=25)
  节点4:     x=150, y=190, w=160, h=40
  节点5:     x=400, y=190, w=160, h=40
背景-数据层: x=50,  y=270, w=700, h=80   (z_index=0, opacity=25)
  节点6:     x=250, y=290, w=160, h=40
```

### 行列对齐（matrix 模式）

适合对比表、功能矩阵。固定行列间距。

**计算方法**：

```
cell_width = 150, cell_height = 40
col_gap = 20, row_gap = 15

第 (r, c) 个单元格:
  x = start_x + c * (cell_width + col_gap)
  y = start_y + r * (cell_height + row_gap)
```

**示例：3x4 对比表**：

```
表头行 (r=0): fill_color="#1F2329", textColor="#FFFFFF"
  (0,0): x=100, y=80,  w=150, h=40  "维度"
  (0,1): x=270, y=80,  w=150, h=40  "方案 A"
  (0,2): x=440, y=80,  w=150, h=40  "方案 B"

数据行 (r=1):
  (1,0): x=100, y=135, w=150, h=40  "性能"
  (1,1): x=270, y=135, w=150, h=40  "高"
  (1,2): x=440, y=135, w=150, h=40  "中"

数据行 (r=2): y=190 ...
数据行 (r=3): y=245 ...
```

### 树状展开（tree 模式）

适合组织架构。根居中，子节点向下均匀展开。

**计算方法**：

```
根节点: x = canvas_center - width/2, y = top
子节点层: y = parent_y + parent_height + layer_gap

子节点水平均分:
  total_width = n * node_width + (n-1) * sibling_gap
  start_x = parent_center_x - total_width / 2
  child[i].x = start_x + i * (node_width + sibling_gap)
```

**示例：1 根 + 3 子 + 6 孙**（画布宽 800px）：

```
根:      x=300, y=50,  w=200, h=50
子1:     x=80,  y=170, w=160, h=40
子2:     x=320, y=170, w=160, h=40
子3:     x=560, y=170, w=160, h=40
孙1-1:   x=30,  y=280, w=120, h=40
孙1-2:   x=170, y=280, w=120, h=40
孙2-1:   x=290, y=280, w=120, h=40
孙2-2:   x=430, y=280, w=120, h=40
孙3-1:   x=530, y=280, w=120, h=40
孙3-2:   x=670, y=280, w=120, h=40
```

### 岛屿式（island 模式）

多个独立模块，各自占据一块区域，模块间用连线连接。

**方法**：
1. 将每个模块视为一个矩形区域
2. 先确定模块的大致位置（上下左右关系）
3. 模块内部按行排列子节点
4. 模块之间留足连线空间（>= 80px）

```
模块 A 区域: x=50,  y=50,  占 300x200
模块 B 区域: x=430, y=50,  占 300x200  (与 A 间距 80px)
模块 C 区域: x=240, y=330, 占 300x200  (与上方间距 80px)
```

### 自由定位（free 模式）

节点位置本身承载信息（坐标轴、角度、方位）。需要数学计算坐标。

适合鱼骨图、柱状图等需要精确坐标的图表。建议：
1. 先确定坐标系（原点、轴方向）
2. 用数学公式计算每个节点位置
3. 复杂场景考虑写脚本生成 JSON

## 背景分区技巧

用低透明度大矩形作为视觉分区，让读者一眼看出分组：

**规则**：
- z_index: 0-1（最底层）
- fill_opacity: <= 25（不遮挡上层节点）
- border_style: solid, border_width: narrow
- border_opacity: 30-50（弱化边框）
- 分区包裹所有子节点 + padding 20-30px

**不同分区用不同颜色**（从 style.md 色板选取）：

```
接入层背景: fill_color="#F0F4FC", border_color="#5178C6"
服务层背景: fill_color="#EAE2FE", border_color="#8569CB"
数据层背景: fill_color="#DFF5E5", border_color="#509863"
```

## 居中对齐技巧

**整图居中**：
```
content_width = max(所有节点 x + width) - min(所有节点 x)
offset = (canvas_width - content_width) / 2 - min(所有节点 x)
每个节点 x += offset
```

**层内居中**（N 个等宽节点）：
```
total = N * node_width + (N-1) * gap
start_x = (canvas_width - total) / 2
```

**单个节点在区域内居中**：
```
node.x = area.x + (area.width - node.width) / 2
node.y = area.y + (area.height - node.height) / 2
```

## 常见错误

错误：所有节点 y=0，堆在一条线上
正确：每层 y 递增 80-120px

错误：节点间距太小（< 20px），连线箭头看不清
正确：有连线的节点间距 >= 60px

错误：背景分区和节点 z_index 相同，节点被遮挡
正确：背景 z_index=0，节点 z_index=10

错误：手写复杂坐标导致节点重叠
正确：用公式计算（或写脚本），保持间距一致
